# 投資分析方法定義書（METHOD）

**現行バージョン: v1.1**（2026-07-05 改訂）

このファイルが「最も儲かる銘柄を見つける」ための唯一の正典。選定は必ずこの手法に従い、
答え合わせ（7日/30日/90日後）の結果によってのみ改訂される。改訂履歴は末尾に必ず残す。

---

## 1. 目的と成功の定義

- **目的**: 日本株・米国株それぞれから、今後上昇が期待できる銘柄を**20銘柄ずつ**選ぶ。
- **成功の定義（答え合わせの判定基準）**:
  - 一次判定: 対ベンチマーク超過リターン（JP=日経225、US=S&P500）がプラス
  - 二次判定: 絶対リターンがプラス
  - 検証時点: 選定から **7日 / 30日 / 90日**（補助的に180日）
- 発注はしない。シグナル生成・記録・検証まで（フォワードテスト）。

## 2. 母集団（ユニバース）

| 市場 | 範囲 | 取得手段 | 更新目安 |
|---|---|---|---|
| 日本 | 東証全銘柄（TOKYO PRO除外、約3,700） | J-Quants API v2 | 月1回 |
| 米国 | S&P500+400+600（約1,500） | Wikipedia構成銘柄リスト | 月1回 |

## 3. スコアリング（2軸・各0-100点）

### 3.1 テクニカル軸（配点50%）
`chart-analysis`スキルで実測検証済みのTier1条件をスコア化:
- ゴールデンクロス発生直後（50日MA > 200日MA 転換）
- 20日モメンタム（上昇率）
- 52週高値圏（高値からの距離が近い）
- RSI適温帯（おおむね45〜70。過熱・過冷を減点）
- **流動性フィルタ（必須）**: 20日平均売買代金 JP≥1億円/日、US≥$5M/日

### 3.2 ファンダメンタル軸（配点50%）
直近決算から、固定閾値ではなく**母集団内パーセンタイル**で採点:
- 収益性: ROE、営業利益率
- 安全性: 自己資本比率
- 割安性: PER、PBR（低いほど高得点）
- 欠損指標は計算から除外（0点扱いにしない）

### 3.3 統合
`総合スコア = テクニカル×0.5 + ファンダ×0.5` → 市場ごとに上位40銘柄を候補プールとする。

## 4. 最終選定（LLMの判断領域）

候補プール上位40から各市場20銘柄に絞る。ルール:
1. **セクター分散**: 同一セクター最大4銘柄（選定数の20%上限）
2. **赤信号チェック**（candidate-analystサブエージェントがWeb検索で確認）:
   - 直近の決算ミス・下方修正・不祥事・訴訟
   - 7日以内に決算発表を控える銘柄は警戒（イベントリスク明記の上で採否判断）
   - バイオ等のバイナリーイベント（治験結果待ち）銘柄はリスク明記
3. 赤信号があれば次点繰り上げ。スコア差5点以内なら赤信号のない方を優先。
4. 最終判断はメインエージェント（Fable/Opus）が横断的に行う。

## 5. 記録（選定のたび必須）

- `scripts/enrich_picks.py`で選定銘柄に **PER・PBR・時価総額・次回決算日** を付与
- `snapshot.py`で選定日終値・ベンチマーク終値・検証予定日を固定記録 → `data/snapshots/`
- 選定根拠表（1銘柄1行: PER/PBR/時価総額/次回決算日・テクニカル根拠・ファンダ根拠・リスク）
  → `reports/` とチャット提示

## 6. 答え合わせと改訂の規律

1. 7/30/90日後に `review.py` でリターン・対ベンチ超過・勝率・**スコアと成績の順位相関**を算出
2. ワースト銘柄は loss-analyst サブエージェントが敗因を分類:
   `決算ミス / セクター逆風 / 地合い(市場全体) / 選定時に見えていた赤信号の見落とし / 過熱買い / 不運`
3. 教訓は `LESSONS.md` に日付付きで追記
4. **改訂の規律（過剰適合防止）**:
   - 1回のレビュー結果だけで配点・条件を変えない
   - 同じ教訓が**2回以上のレビューで再現**した場合のみ、method-auditorエージェントの監査を経て改訂
   - 改訂時はバージョンを上げ、末尾の改訂履歴に「何を・なぜ・根拠となったレビュー」を記録
