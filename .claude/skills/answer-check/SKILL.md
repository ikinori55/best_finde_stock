# answer-check（答え合わせ → 敗因分析 → 手法改訂）

## 実行手順

`SCR = C:\Users\iki_n\.claude\skills\stock-screener\scripts`

### 1. 対象確認
```
python scripts/check_due.py
```
期日到来・未実施のレビュー（スナップショット×ラベル）を特定する。

### 2. 事実の計算（Python）
期日到来分それぞれについて:
```
python $SCR/review.py --snapshot data/snapshots/snapshot_YYYYMMDD.csv --label 7d
```
出力: 銘柄別リターン・対ベンチマーク超過・勝率・ベスト/ワースト・スコアと成績のスピアマン順位相関。
出力CSV/JSONは `data/snapshots/` に自動保存される（これが「実施済み」の判定材料になる）。

### 3. 敗因診断（並列サブエージェント）
対ベンチマーク超過がマイナスの銘柄を `loss-analyst` サブエージェントに診断させる。
1体あたり最大5銘柄、複数体を1メッセージで同時起動。
各銘柄に渡すもの: ticker・選定日・選定時スコア・選定根拠（reports/の根拠表から）・
期間リターン・ベンチマークリターン。

### 4. 解釈と教訓化（メインエージェント）
1. 勝率・超過リターン・順位相関から「手法が機能したか」を判定
   - 順位相関がゼロ〜負 → そのスコア軸は機能していない疑い（教訓として記録）
   - 地合い（ベンチ自体の下落）と手法の欠陥を混同しない
2. loss-analystの診断を統合し、`LESSONS.md` に日付付きで追記
   - 過去教訓と同内容なら「★再現」マークを付ける
3. レビューレポートを `reports/review_YYYYMMDD_<label>.md` に保存し、チャットに要約提示:
   `| 銘柄 | リターン | 対ベンチ超過 | 勝敗 | 敗因分類 |` + 総括

### 5. 手法改訂（条件を満たす場合のみ）
改訂条件: **同じ教訓がLESSONS.mdで2回以上のレビューにわたり再現（★再現）**
1. 改訂案を作成（何を・どう変えるか・根拠教訓）
2. `method-auditor` サブエージェントに監査させる
3. 「承認」の場合のみ METHOD.md を改訂し、バージョンを上げ、改訂履歴に記録
4. LESSONS.md の該当教訓に「→ vX.X反映」を記す
差し戻しの場合は改訂せず、監査意見をLESSONS.mdに記録して次回レビューで再検討。

## 禁止事項
- リターンをLLMの暗算・記憶で出すこと（必ずreview.pyの数字を使う）
- 1回のレビュー結果だけでMETHOD.mdを改訂すること
- method-auditorの監査を経ない改訂
- 悪い結果の言い訳に終始し「選定時見落とし」を認めないこと
